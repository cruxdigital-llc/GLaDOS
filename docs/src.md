# Source Tree

`src/` holds the source the compiler assembles into installed workflow text.
Nothing in `src/` is used verbatim at run time ‚Äî the install compiles cores,
kernel fragments, vocabulary partials, and enabled modules against the
project's `glados.yaml` into one canonical output, then emits it through the
chosen runtime adapter.

## `src/kernel/`

The fragments the compiler wraps around every core, and the contracts it
checks against.

| File | Role |
|---|---|
| `preamble.md` | Prepended to every core: create the run record first, record `work.base-sha` before mutating, resolve decision rights. Carries the `{{OPTIMIZE_FOR}}` slot the phase preset fills. |
| `epilogue.md` | Appended to every core: finalize the run record, yield check, commit, publish outcomes to bound channels, release holds. A core defining its own ending fails lint. |
| `state-registry.yaml` | The flat state-key registry ‚Äî every legal `reads:`/`writes:` token with its declared home, plus the outcome-type list. Install fails if an enabled read has no enabled writer. |
| `presets/phases.yaml` | The vendored phase preset table (`nascent`/`evolving`/`production`/`sunset`), the precedence rules, and the phase-untouchable invariants. |

## `src/vocabulary/`

Single-source rule partials, inlined where a core includes them via
`<!-- glados:include vocabulary/<file> -->`:

- `verdicts.md` ‚Äî the one severity scale and verdict vocabulary, with composition rules.
- `panel-roster.md` ‚Äî how the review panel is seated (standing lenses + manifest + feature personas).
- `loop-bounds.md` ‚Äî the bounded-loop rule; bound values resolve from manifest params.
- `git-conventions.md` ‚Äî commit/git defaults, subordinate to the project's own agent instructions.
- `branch-naming.md` ‚Äî branch names and MR targets resolve from `glados.yaml` ‚Üí `branching:`.

## `src/workflows/`

The 15 cores (`kind: core` frontmatter) ‚Äî nothing else lives here. The
permanent v1 aliases are entries in `src/kernel/aliases.yaml`, emitted as
one-line shims by every adapter. See [workflows.md](workflows.md) for the
full tables.

## `src/modules/`

The three compiled modules (`kind: module` frontmatter). The retired v1
module files are deleted; see [modules.md](modules.md).

## `src/personas/`

Persona definitions seatable on review panels and ceremonies. The ceremony
set shipped for `brunch`: `brunch-moderator`, `test-engineer`, `qa`,
`ux-advocate`, `code-cleanliness`, `data-modeler`, `scalability-engineer`,
`security-engineer`, `devops-engineer` ‚Äî alongside `architect` and
`product-manager`. Every install vendors this library into the target's
`.glados/personas/`. Projects add their own by dropping a file in their
`product-knowledge/personas/` directory (searched before the vendored
library) and naming it in the manifest (lane 3 ‚Äî no reinstall).

## `src/templates/`

Boilerplate the install scaffolds (`PROJECT_STATUS.md`) and the plugin `init`
skill copies on demand (`SPEC_LOG.md`, `CLAIMS.md`, `SDA_ROADMAP.md`,
observation/standards/philosophies README stubs).

## `src/overlays/`

The v1 overlay mechanism; **not compiled in v2** ‚Äî kept as a fixture only
(`test-overlay/`). See [overlays.md](overlays.md) for the v2 customization
surface.

## Other top-level directories

- **`bin/`** ‚Äî the installer tooling; `python bin/glados.py install --mode
  <claude|claude-plugin|direct|gemini|antigravity|aistudio>` is the entry
  point. The compiler is under active development; see `bin/README.md`.
- **`skills/`** ‚Äî Claude Code skill wrappers (thin entrypoints delegating to
  workflows), regenerated by the claude-plugin adapter at compile time.
- **`docs/`** ‚Äî this documentation, [V2_STRATEGY.md](V2_STRATEGY.md), persona
  and overlay guides, and `docs/standards/` (the SDA Standard v1.0 and GLaDOS
  Profile v1.0).
- **`glados.yaml.example`** ‚Äî the manifest template projects copy to their
  repo root.
- **`PLAYBOOK.md`** ‚Äî team-adoption guidance.
